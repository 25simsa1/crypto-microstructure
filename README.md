# crypto-microstructure

A market microstructure research platform built on live exchange
capture across three venues: one night of Binance.US (top-20 book
snapshots ~1/s plus the trade tape) and a multi-day dual-venue Kraken +
Coinbase capture (book, then trades added as separate loggers), all
turned into a typed, tested, reproducible research pipeline.

The point is not a trading strategy. The central result is a
**pre-registered, cross-venue replication**: night one's headline —
SOL trade-sign order-flow memory — was frozen in `REPLICATION.md`
*before* the new data was touched, then re-run on Kraken and Coinbase
and **replicated on both** (SOL ACF(1–5) +0.26 / +0.33, Ljung–Box p~0,
vs night one's +0.15). This is independent confirmation of a *known*
effect (order-flow autocorrelation; Cont et al.), not a new discovery —
the contribution is the rigor: a frozen protocol, an empirically
validated trade-side convention without which Coinbase's result would
have silently inverted, and a print-fragmentation confound caught in a
bug hunt (collapsing it, Kraken lands at +0.159, essentially night one's
+0.15 — robust, not an artifact).

Every other headline is an honest negative or a guarded one, kept
separate from the replication: **no strategy in the tested family
survives retail fees** (round-trip cost ~1000× the per-trade edge);
cross-venue mid divergence never exceeds a round trip; and the earlier
book-only night-two analyses (a tick-sign analogue, the Epps curve)
returned a mixed verdict — reported as-is, with one venue voided by a
data-quality freeze the pipeline caught itself.

The point is the *methodology*: clean data layers, hand-verified feature
definitions, statistics that punish multiple testing, a backtester that
physically cannot look ahead, a pre-registered replication reported
either way, and a documented progression (book-only "not testable" →
trade capture added → replicated) left intact rather than retconned.

## Architecture

```
            live capture (runs unattended)
  ┌─────────────────────────────────────────────┐
  │  logger.py            trades_logger.py      │
  │  book snapshots 1/s   every trade           │
  │        └──────┬──────────────┘              │
  │               ▼                             │
  │  data/book-*.jsonl.gz, trades-*.jsonl.gz    │  hourly-rotated, gzip,
  └───────────────┬─────────────────────────────┘  flushed every write
                  ▼
        src/microstructure/
  ┌─────────────────────────────────────────────┐
  │ ingest.py    typed parsing, live-file safe  │
  │ quality.py   gaps / crossed books / report  │
  │ parquet.py   symbol-hour partitioned store  │
  │ features.py  microprice, OFI, imbalance,    │
  │              RV / Parkinson / bipower       │
  │ anomaly.py   spread/depth/vol episode flags │
  │ backtest.py  event replay, walk-the-book    │
  │              fills, deflated Sharpe         │
  │ analysis.py  shared plotting/loading        │
  └───────────────┬─────────────────────────────┘
                  ▼
              scripts/            output/
  build_data.py ──────────────▶ data_quality.md
  analysis_*.py (7 studies) ──▶ *.png + *.md
  run_backtest.py ────────────▶ backtest.md
  build_findings.py ──────────▶ ../FINDINGS.md
```

## Quick start

```bash
python3 -m venv .venv && .venv/bin/pip install -e '.[dev]'
make all     # parquet + quality report + 7 studies + backtest + FINDINGS.md
make check   # ruff + mypy --strict + pytest
```

The loggers (`logger.py`, `trades_logger.py`, stdlib + `websockets` only)
run separately: `caffeinate -i python3 logger.py`.

## What's inside

| module | what it does | the detail that matters |
|---|---|---|
| `ingest.py` | jsonl.gz → typed dataclasses | tolerates truncated gzip tails from live files |
| `quality.py` | validation | gaps, crossed books, backwards timestamps — known before any analysis |
| `parquet.py` | columnar store | idempotent; only rewrites partitions that grew |
| `features.py` | feature library | every formula stated in the docstring, unit-tested against hand-computed numbers |
| `anomaly.py` | episode detection | trailing-robust thresholds tuned on real tick-quantized data, not textbook z-scores |
| `backtest.py` | event-driven replay | decisions at snapshot *t* execute against snapshot *t+1* — lookahead is structurally impossible |

## Honesty constraints

- Imbalance/return regressions use Newey–West errors and a Bonferroni
  bar across all 27 hypotheses tested.
- The backtest pays taker fees, crosses the spread, and cannot fill more
  than displayed size; its Sharpe is deflated for the 9 configs tried
  and compared against 50 random-strategy seeds.
- Every report carries a data-span note; `FINDINGS.md` opens with what
  one night of single-venue snapshots cannot support.

## Results

Full detail in [FINDINGS.md](FINDINGS.md) (regenerated by `make all`);
headlines:

- **H1 — SOL trade-sign memory: REPLICATED cross-venue.** Kraken
  ACF(1–5) +0.26, Coinbase +0.33, both Ljung–Box p~0 (night one: +0.15).
  Robust to print fragmentation (collapsed: +0.159 / +0.291). Confirms a
  known effect on independent venues; not a novel finding.
- **Economic null — strategies lose to fees.** Best backtest config has a
  deflated Sharpe of 0.00, inside the random-strategy null; cross-venue
  divergence never beats a round trip. Statistically-real ≠ tradable.
- **Night-two book-only (dated, preserved):** tick-sign analogue did not
  replicate; Epps curve replicated on Kraken, criterion-failed on
  Coinbase. Mixed verdict, reported as-is.

Caveats stay attached to each result: 1 s book cadence, retail-venue
data, and a trade-side convention (`SIDE_CONVENTION.md`) the H1 verdict
depends on.
